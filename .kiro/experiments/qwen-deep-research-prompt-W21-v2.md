# 通义千问 Deep Research — W21 改进版 Prompt (v2)

**改进目标**：上一版 Qwen DR 把"周度热点雷达"做成了"长期教育图谱"。
v2 加 5 处约束把它拽回"本周事件 + 本周讨论"维度。

**用法**：
1. 打开 https://chat.qwen.ai
2. 开 **深度研究 / Deep Research** toggle
3. **完整复制**下面 `### PROMPT START ###` 到 `### PROMPT END ###` 之间的内容
4. **必须**：替换 `[start_date]` `[end_date]` `[week_label]` 为实际值
5. 等 8-15 分钟
6. 拷出 markdown，发给我做 URL probe 校验

---

### PROMPT START ###

我是亚马逊"账户健康"团队的产品经理，每周需要一份**本周时效性的事件雷达**，
不是长期合规教育报告。

**任务**：制作 [week_label]（[start_date] 至 [end_date]）中国跨境卖家社区
**周度热点雷达**。

⚠️ **重点：这是事件雷达，不是合规科普**。我要的是：
- 本周哪个具体新闻 / 政策更新 / 平台动作 / 卖家集体事件在引爆讨论
- 本周哪个具体卖家发了求救帖 / 维权帖 / 经验帖
- 本周哪个新规执行进入倒计时 / 突然加严

我**不要**："长期议题图谱"、"卖家普遍误区分析"、"教育方案矩阵"、
"卖家分层培训建议"——这些都是产品衍生品，不是雷达本身。

如果某个 topic 没有 [start_date] 到 [end_date] 之间的**具体事件触发点**，
就**不要列**——宁愿少 topic，不要凑教育内容。

---

**研究范围**

只关注**亚马逊全球开店中国卖家**的相关讨论。三个独立模块：

模块 A. **账户健康 / 账户合规**
账户被封号 / 停用 / 警告，绩效绿黄红区变化，二审 / KYC / 视频验证，
关联封号，AHR 规则更新，AHA 项目，Seller Challenge 新规等。

模块 B. **Listing 上下架 / 商品合规**
Listing 下架 / 屏蔽 / 冻结，侵权投诉，内容合规，变体清理，
类目违规，禁限售商品，CPSC / GPSR / 能效标签等强制合规倒计时。

模块 C. **支持工具与申诉渠道反馈**
卖家对 AHA / AHR / Call Me Now / Seller Challenge / 
Account Health Dashboard / Seller Assistant VA / case 升级渠道
等亚马逊**支持工具的吐槽 / 求助 / 经验**——本周新发生的，不是长期教育。

---

**强制约束（每个 topic 必须满足）**

每个 topic 必须有 **at least one** 以下"本周事件锚点"，否则不入榜：

1. **本周具体日期的平台公告 / 政策动作**（例：5/15 Amazon 宣布 SPS 团队
   裁员、5/20 CPSC eFiling 倒计时进入两个月）
2. **本周具体日期的卖家社区帖子**（例：5/22 知无不言一个卖家发帖说账号
   突然被封）
3. **本周具体执行 / 倒计时的合规节点**（例：5/31 变体评论拆分新规全类目
   落地最终期限）
4. **本周突发的执法行动 / 批量动作**（例：5/19-5/24 巴西 / 加拿大站附属
   账号批量收 KYC 邮件）

**不接受**：
- "AHR 体系焦虑"——这是过去几个月持续话题，没本周触发就不算事件
- "IP 侵权是平台老大难"——这是行业常识，不是雷达
- "税务合规是底线"——同上

---

**Topic 命名风格要求**

正面例子（具体事件）：
- "5/20 CPSC eFiling 进入 7/8 强制倒计时，跨境卖家爆发自查潮"
- "5/15 Amazon 确认 SPS 卖家服务团队再裁员，卖家担忧申诉时效"
- "5/22 知无不言爆发 Risk-Shield 秒封讨论，AHR 满分卖家被冻结"

错误例子（综述/教育，⚠️不要这种）：
- "重复违规触发 AHR 体系的连锁反应"
- "知识产权侵权"
- "税务合规问题"

---

**信源优先级**

中文社区为主：
- 卖家社区：知无不言（wearesellers）、卖家之家、AMZ123、跨境圈、
  亿恩网、雨果网论坛
- 行业媒体：雨果网、亿恩网、跨境眼、跨境头条、网经社、AMZ123 资讯、
  钛媒体、新浪财经
- 服务商内容：万门、CIO 时代等代运营 / 法务 / 合规服务商
- KOL：知乎"亚马逊"、微信公众号、小红书、B 站、抖音跨境频道

英文补充（仅当本周确有相关报道）：
- Reuters、Bloomberg、CNBC、Amazon Seller Central forum

**每个 topic 必须输出**

```
### N. [本周事件性标题]
- **本周事件锚点**：[必填，引用上述 4 类锚点之一] 
- **本周热度估计**：[使用如下加权声量公式]
  总声量 = 论坛帖文数×1.0 + 服务商文章数×2.0 + 媒体文章数×4.0 + KOL视频/文章数×5.0
  示例: 论坛 8 + 服务商 3 + 媒体 5 + KOL 2 = 8+6+20+10 = 44
- **卖家情绪**：负面 / 焦虑 / 困惑 / 中性 / 正面
- **Severity**: High / Medium / Low
  - High = 影响店铺生死（直接封号 / 资金冻结 / 大面积下架）
  - Medium = 影响销售但可挽救（Listing 下架但可申诉）
  - Low = 流程低效 / 体验糟糕但短期内不致命
- **本周 case**（如有）：哪个卖家、店铺规模、时间线、当前状态、来源 URL + 日期
- **量化观察**（如有）：具体数字 + 来源（例: 资金冻结 60-90 天）
- **背景说明**（如适用）：长期议题的历史背景，**明确和"本周新发生的讨论"分开**
- **misconception**（如适用）：本周讨论中浮现的卖家常见误解
- **本周来源**（≥3条，每条带：平台名、URL、发布日期 YYYY-MM-DD）：
  - [平台名] [日期] [URL]
  - ...
```

**禁止事项（重要）**

- ❌ 严禁编造引用、URL、卖家言论、case
- ❌ 严禁把 [start_date] 之前的旧事件包装成本周热点
- ❌ 严禁靠"这是普遍痛点"这种概括性套话填空
- ❌ 严禁加"教育方案矩阵 / 培训主题 / 卖家分层建议"等产品衍生章节
- ❌ 严禁用 Topic 名命成"知识产权侵权"、"税务合规"这种长期议题——
  topic 命名必须是本周具体事件
- ❌ 严禁把 quote 写成"代表性合成语"——每条 quote 必须是真实卖家
  在本周说的，配真实 URL + 日期。如果找不到真实 quote，**该字段就留空**，
  不要靠合成语填充

请仔细规划研究计划，**特别重点确认你的研究计划聚焦在 [start_date] 至 
[end_date] 这 7 天内的事件**，然后开始研究。
