# Gemini Deep Research — W21 实验 Prompt

**用法**：
1. 打开 https://gemini.google.com（你的 Pro 账号）
2. 点 Deep Research（左下"Tools"里）
3. **完整复制**下面 `### PROMPT START ###` 到 `### PROMPT END ###` 之间的内容粘进去
4. **必须**：先把 `[start_date]` `[end_date]` `[week_label]` 替换成你 W21 实际的时间，例如：
   - `[start_date]` → `2026-05-19`
   - `[end_date]` → `2026-05-26`
   - `[week_label]` → `W21`
5. Deep Research 会先生成一个 plan（搜索计划）问你确认。点 "Start research"
6. 等 8-15 分钟跑完
7. Export 报告（顶部 "Export to Docs" 或直接复制）

### PROMPT START ###

我是亚马逊"账户健康"团队的产品经理，每周需要一份中国跨境卖家社区的情报雷达。
请帮我做一份本周（[start_date] 至 [end_date]，标识 [week_label]）的中文跨境
电商卖家社区情报研究报告。

**研究范围**

只关注**亚马逊全球开店中国卖家**的相关讨论。三个独立模块，分开输出：

模块 A. **账户健康 / 账户合规**
账户被封号 / 停用 / 警告，绩效绿黄红区变化，二审 / KYC / 视频验证，
关联封号，账户健康评分（AHR）规则更新，Account Health Assurance (AHA) 
项目，Seller Challenge 新规等。

模块 B. **Listing 上下架 / 商品合规**
Listing 被下架 / 屏蔽 / 冻结，侵权投诉（专利 / 商标 / 版权 / 形象侵权），
内容合规（图片 / 标题 / 描述违规），变体清理，类目违规，禁限售商品。

模块 C. **支持工具与申诉渠道反馈**
卖家对 Amazon 自己的支持工具的吐槽 / 求助 / 经验分享，包括：
Account Health Assistant (AHA) / Account Health Rating (AHR) / 
Call Me Now / Seller Challenge / Account Health Dashboard / 
Seller Assistant VA / case 升级渠道效率等。

**研究方法要求**

1. **信源优先级**（请尽量覆盖）：
   - 卖家原生社区：知无不言、卖家之家、AMZ123、跨境圈、亿恩网、雨果网
     论坛部分
   - 行业媒体：雨果网、亿恩网、跨境眼、跨境头条、网经社、AMZ123 资讯
   - 服务商内容：万门、CIO 时代等代运营 / 法务 / 合规服务商发布
   - KOL / 自媒体：知乎 "亚马逊"、微信公众号、小红书、B 站跨境频道
   - 英文补充：Amazon Seller Central forum、Reddit r/AmazonSeller（仅在
     讨论中国卖家相关时）

2. **时间约束**：所有引用必须来自 [start_date] 至 [end_date] 之间发布的
   内容。**严禁**把 2024 年、2025 年的旧事件 / 旧帖子作为本周热点呈现。
   如果某个话题的最新讨论是 [start_date] 之前的（哪怕只早一周），请明确
   标注"截至 [start_date] 之前的最新一次讨论"，不能算作本周热点。

3. **诚实优先于凑数**：如果某个模块本周真实信号只有 2-3 条，就只输出
   2-3 条。每个模块**目标 Top 5**，但不要为了凑 5 条把弱信号 / 历史话题
   塞进来。

4. **每个 topic 必须给我**：
   - **主题名称**（一句话，直击痛点 / 事件，例如"二审视频验证审核趋严
     导致老账号批量被拒"）
   - **本周热度估计**（discussions / posts 数量 + 来自几个不同信源 + 
     卖家情绪倾向：负面 / 焦虑 / 困惑 / 中性）
   - **本周关键事件 / 触发点**（如果有具体日期 / 具体新闻）
   - **本周卖家原话引用 ≥ 2 条**：每条带 (a) 来源 URL；(b) 发布日期
     YYYY-MM-DD；(c) 来源平台名（知无不言 / 卖家之家 / 雨果网 ...）
   - **本周具体 case 引用 ≥ 1 条**（如果有的话）：哪个卖家、店铺规模、
     时间线、当前状态。来源 URL + 日期。
   - **量化观察 ≥ 1 条**（如果有）：具体数字 + 来源。例如"45 天资金
     冻结期"、"提交三次资料被打回"。
   - **背景说明**：如果这是个长期议题（非本周首发），请用一段简短
     "背景说明"或"政策参考"把历史交代清楚——但要明确和"本周新发生的
     讨论"分开。
   - **Severity 严重度判断**：High / Medium / Low（你的判断）
     - High = 影响店铺生死，例如直接封号 / 资金冻结
     - Medium = 影响销售但可挽救，例如 Listing 下架但可申诉
     - Low = 流程低效 / 体验糟糕但短期内不致命

5. **misconception 字段（仅在适用时）**：本周有没有发现卖家常见
   误解（卖家以为政策是 X，实际政策是 Y），如果有，请单独整理。

**禁止事项**
